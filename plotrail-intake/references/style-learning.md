# Style Learning

Use this reference when the user asks to learn author style, compare original and revised drafts, or build a reusable editing profile.

## Integrity Rule

Do not claim a line-level editing pattern unless both original and revised versions exist. If only final prose exists, produce a `Style Snapshot`, not an `Author Editing Profile`.

Do not imitate a named living author's distinctive style. If the user provides admired authors, extract transferable craft targets such as pacing, restraint, dialogue pressure, sensory texture, chapter hooks, or information release. Keep the user's own voice as the highest-priority style source.

## Evidence Sources

Look for:

- original and revised chapter pairs
- git history
- `drafts_original/`, `backup/`, `old/`, or exported versions
- review notes that quote old lines
- accepted chapters compared with later rough drafts
- chapter contracts and final chapters

If exact originals are unavailable, say so and triangulate carefully.

## Style Snapshot Dimensions

- paragraph length and rhythm
- sentence length and clause density
- dialogue-to-narration ratio
- interiority versus external action
- sensory detail type
- explanation density
- metaphor or image habits
- scene opening pattern
- chapter ending hook pattern
- taboo phrases or generic prose to avoid
- AI-flavored prose patterns to remove
- optional craft anchors requested by the user, stated as techniques rather than imitation targets

## Editing Profile Dimensions

When true original/revised comparisons exist, infer:

- what the author tends to keep
- what the author tends to cut
- how the author clarifies causality
- how the author sharpens character agency
- how exposition is moved or reduced
- how hooks are upgraded
- what emotional or moral texture is added
- what model-like prose is removed
- how the revision avoids flattening the author's voice into generic polish

## Output

For final prose only:

```markdown
# Style Snapshot

## Keep
## Avoid
## Voice Markers
## Scene Requirements
## Hook Preferences
## Confidence And Limits
```

For original/revised comparisons:

```markdown
# Author Editing Profile

## Evidence Used
## Keep
## Avoid
## Revision Patterns
## Drafting Requirements
## Review Checklist
```

When working in a PlotRail project, save durable results to:

```text
memory/author_editing_profile.md
```

Keep the profile operational. Avoid vague labels unless backed by concrete behaviors.
